## 3. 실행 방법

### 방법 A: run.bat 더블클릭 (권장)
탐색기에서 `run.bat` 파일을 더블클릭하면 자동으로:
- yt-dlp 최신 버전 업데이트
- Cloudflare Tunnel 연결 (설치된 경우)
- 서버 시작

### 방법 B: 터미널에서 직접 실행
```bash
.\run.bat
```

### 실행 성공 시 화면:
```
==================================================
  YouTube Downloader v2.0
  Server + Cloudflare Tunnel
==================================================

[1/3] yt-dlp 업데이트 확인 중...
[2/3] Cloudflare Tunnel 연결 중...

==================================================
  외부 접속 주소: https://xxxx.trycloudflare.com
==================================================

[3/3] 서버를 시작합니다...
  로컬 주소: http://localhost:5000
```

## 4. 사용 방법

### 4-1. 영상 다운로드 (MP4)
1. 브라우저에서 `http://localhost:5000` 접속
2. 유튜브 URL 붙여넣기
3. **VIDEO** 모드 선택
4. **「매체 분석 및 저장 →」** 클릭
5. 분석이 완료되면 **「파일 디바이스에 저장」** 클릭

### 4-2. 음원 다운로드 (M4A)
1. 위와 동일하되 **AUDIO** 모드 선택
2. AAC 192kbps 품질로 자동 변환됨

### 4-3. 다운로드 파일 위치
- **로컬 사용 시**: 프로젝트 폴더 내 `downloads/` 디렉토리
- **외부 접속 시**: 접속한 기기로 직접 다운로드됨

## 5. 외부 접속 (Cloudflare Tunnel)

`cloudflared`가 설치되어 있으면, `run.bat` 실행 시 자동으로 외부 접속 URL이 생성됩니다.

### 접속 방법
1. 터미널에 표시된 `https://xxxx.trycloudflare.com` 주소를 복사
2. 스마트폰이나 다른 PC의 브라우저에서 접속
3. 동일한 다운로더 화면이 표시됨

### 주의사항
- URL은 `run.bat`을 **실행할 때마다 변경**됩니다
- **PC가 켜져 있고 서버가 실행 중인 동안**에만 접속 가능
- `Ctrl+C`로 서버를 종료하면 터널도 자동으로 닫힘

## 6. 자주 묻는 질문 (FAQ)

### Q. "Sign in to confirm you're not a bot" 에러가 나요
- **원인**: 유튜브가 자동화된 봇으로 의심하여 차단
- **해결**: 로컬 PC에서 `run.bat`으로 실행하면 가정 IP를 사용하므로 해결됨. 클라우드(Render 등)에서는 이 문제가 발생할 수 있음

### Q. 다운로드한 영상이 재생되지 않아요
- **원인**: AV1, VP9 등 비호환 코덱
- **해결**: v2.0에서는 자동으로 H.264(avc1) + AAC(mp4a) 코덱을 우선 선택하여 모든 기기에서 재생 가능

### Q. "WinError 32: 파일을 사용 중" 에러가 나요
- **원인**: 서버가 여러 개 실행되어 파일 충돌
- **해결**: 터미널 창을 모두 닫고 `run.bat`을 하나만 실행하세요

### Q. 외부 접속 URL을 고정하고 싶어요
- Quick Tunnel은 매번 URL이 바뀝니다
- 고정 URL이 필요하면 [Cloudflare Zero Trust](https://one.dash.cloudflare.com/) 에서 Named Tunnel을 설정하세요 (무료)

### Q. 서버를 종료하려면?
- 터미널에서 `Ctrl+C`를 누르면 서버와 터널이 모두 종료됩니다
